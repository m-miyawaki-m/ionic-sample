# Androidアプリ検討チェックリスト

Webアプリにはない「Androidアプリだからこそ検討すべき項目」を網羅的に整理したチェックリスト。
倉庫在庫管理アプリの要件定義時に、各項目を「必要/不要/将来検討」で判断する。

---

## 使い方

各項目を以下の3段階で評価する:

- **必要** — 初期リリースで対応する
- **将来** — 初期リリースでは不要だが、将来対応する可能性がある
- **不要** — このアプリでは対応しない

---

## 次のステップ

1. 上記チェックリストの各項目を関係者とレビューし「必要/将来/不要」を判定する
2. 「必要」と判定された項目を要件定義書に反映する
3. 判定結果をもとに設計書・実装計画を更新する
4. サンプルページで実装パターンを確認し、UIデザインを確定する
