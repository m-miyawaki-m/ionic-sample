# Androidアプリ検討チェックリスト

Webアプリにはない「Androidアプリだからこそ検討すべき項目」を網羅的に整理したチェックリスト。
倉庫在庫管理アプリの要件定義時に、各項目を「必要/不要/将来検討」で判断する。

---

## 使い方

各項目を以下の3段階で評価する:

- **必要** — 初期リリースで対応する
- **将来** — 初期リリースでは不要だが、将来対応する可能性がある
- **不要** — このアプリでは対応しない

**Ionic対応**の凡例:

- **◎ 標準** — Ionic/Capacitor の公式プラグインで対応可能。すぐ使える
- **○ プラグイン** — コミュニティプラグインまたはnpmパッケージで対応可能
- **△ カスタム** — Capacitor Local Plugin（Java）を自作して対応
- **▲ 制限あり** — Ionic（WebView）では制限がある。ネイティブ実装が必要な部分あり
- **× 不可** — Ionic/Capacitor では対応不可。フルネイティブのみ

## 用語集

チェックリスト内で使われる専門用語の解説。

| 用語 | 説明 |
|------|------|
| **AAR** | Android Archive。Androidライブラリの配布形式。Java/Kotlinコードとリソースをパッケージ化したもの。`.aar` ファイルとして提供される。このプロジェクトではSP2スキャナのSDKがAAR形式 |
| **Capacitor** | Ionicのネイティブランタイム。Web（TypeScript）からAndroid/iOSのネイティブ機能にアクセスするためのブリッジ。プラグインを通じてカメラ、GPS、バイブレーション等を利用できる |
| **Local Plugin** | Capacitorプラグインの一種。npmパッケージとして公開せず、プロジェクト内（`android/app/src/`）に直接Javaコードを配置する方式 |
| **WebView** | Android内蔵のブラウザエンジン（Chrome系）。Ionicアプリはこの中で動作する。HTML/CSS/JSを描画するが、ネイティブAPIには直接アクセスできないためCapacitorが必要 |
| **BLE** | Bluetooth Low Energy。省電力のBluetooth通信規格。スキャナ等の周辺機器との接続に使用される |
| **NFC** | Near Field Communication（近距離無線通信）。数センチの距離でICタグを読み書きする技術。Suica等でも使われている |
| **GPS** | Global Positioning System。端末の緯度・経度を取得する衛星測位システム。屋内では精度が落ちるため、WiFi/BLEビーコンと併用することもある |
| **Haptics** | 触覚フィードバック。端末を振動させてユーザーに操作結果を伝える技術。Capacitorの`@capacitor/haptics`で利用 |
| **TTS** | Text-to-Speech（テキスト読み上げ）。テキストデータを音声に変換して出力する技術 |
| **プッシュ通知** | アプリがバックグラウンド（画面に表示されていない状態）でもサーバーからメッセージを受け取り、端末に通知を表示する仕組み。Firebase Cloud Messaging（FCM）等を利用 |
| **バッジ** | アプリアイコンの右上に表示される数字。LINEの未読数のようなもの。未処理件数の通知に使う |
| **トースト** | 画面下部に一時的に表示される短いメッセージ。数秒で自動的に消える。Androidの標準的なUI要素 |
| **ダイアログ（アラート）** | 画面中央に表示される確認用のポップアップ。「OK」「キャンセル」等のボタン付き。ユーザーが操作するまで消えない |
| **スプラッシュスクリーン** | アプリ起動時に一瞬表示される画面。企業ロゴ等を表示し、アプリの読み込み完了を待つ |
| **キオスクモード** | 端末を1つのアプリ専用機にするモード。ホーム画面に戻れず、他のアプリを起動できない。業務専用端末で使用 |
| **MDM** | Mobile Device Management（モバイルデバイス管理）。企業が複数の端末を一括で管理するためのシステム。アプリ配布、設定変更、リモートワイプ等をサーバーから実行 |
| **リモートワイプ** | 端末を紛失した場合に、遠隔操作で端末内のデータを消去する機能。情報漏洩対策 |
| **証明書ピンニング** | アプリが通信する際に、特定のSSL証明書のみを信頼する設定。中間者攻撃（通信の盗聴・改ざん）を防ぐ |
| **ディープリンク** | 特定のURLをタップした時に、ブラウザではなくアプリの特定画面を直接開く仕組み。例: `warehouse://inventory/ITEM-001` |
| **Share Intent** | Androidの共有機能。「共有」ボタンを押すと、メール、LINE等の他アプリにデータを送信できる |
| **SQLite** | 軽量なRDB（リレーショナルデータベース）。端末内にファイルとしてデータを保存。サーバー不要 |
| **オフラインキュー** | ネットワーク切断中に発生した操作（登録・更新等）をローカルに溜めておき、ネットワーク復帰時にサーバーに一括送信する仕組み |
| **仮想スクロール** | 数千件のリストを表示する際、画面に見えている部分だけをDOMに描画する技術。メモリ消費を大幅に削減 |
| **プルトゥリフレッシュ** | リスト画面を下に引っ張る操作でデータを再読み込みする、モバイルアプリの標準的なUI操作 |
| **アコーディオン** | 見出し部分をタップすると中身が展開/折りたたみされるUI。限られた画面領域で多くの情報を整理できる |
| **セグメント** | 横に並んだボタン群で表示内容を切り替えるUI。タブに似ているが、より小さくコンパクト |
| **ステッパー** | 数値入力で「+」「-」ボタンで値を増減するUI。スライダーと異なり正確な値を入力できる |
| **dp** | Density-independent Pixels。Androidの画面密度に依存しないサイズ単位。48dpは指で押しやすい最小サイズとされる |
| **TalkBack** | Android標準のスクリーンリーダー（画面読み上げ機能）。視覚障害のあるユーザー向けのアクセシビリティ機能 |
| **フォアグラウンド/バックグラウンド** | フォアグラウンドは画面に表示されている状態。バックグラウンドはホームに戻った等で見えない状態。Webアプリではタブを閉じれば終了するが、Androidアプリはバックグラウンドでも生き続ける |

## 6. データ管理・オフライン

ネットワークが不安定な倉庫環境でのデータ管理。

| # | 項目 | 説明 | 倉庫アプリでの用途例 | Ionic対応 | 対応方法 | 判定 |
|---|------|------|---------------------|----------|---------|------|
| 6-1 | **オフライン対応** | ネットワーク切断時もアプリが動作 | 倉庫の電波が弱いエリアでの作業継続 | ○ プラグイン | SQLiteにデータ保存 + ネットワーク復帰時に同期。設計が必要 | |
